MCP — il protocollo standard per i tool

MCP sta ai tool come USB sta ai dispositivi. Un connettore standard, non un nuovo dispositivo.

Cos'è (e cosa non è)

  • Non è un nuovo tipo di tool, né cambia la meccanica del tool call
  • È un protocollo che standardizza discovery ("quali tool offri?") e invocazione (formato uniforme)
  • Dopo il discovery, le tool definition finiscono nel system prompt e restano lì per tutta la sessione

Perché conta

  • Riduce il vendor lock-in: stessa integrazione su qualunque agente compatibile
  • Centinaia di server pubblici: GitHub, Linear, Notion, Gmail, DB — layer degli app-store per agenti
  • Se un'integrazione AI proprietaria non espone MCP, è vendor lock-in.

  Il trabocchetto

Ogni server MCP connesso paga token nel contesto: le tool definition sono sempre nel system prompt. Connetterne 10 "nel caso servissero" può saturare metà del contesto prima che l'agente inizi a lavorare. Quando un prodotto dichiara "supporta 50 MCP", chiedere quali si attivano di default.
