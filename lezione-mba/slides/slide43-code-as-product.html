Scala

Stesso pattern del code as tool (scrivi-esegui-verifica), moltiplicato per un progetto reale: molti file, dipendenze, test suite, git, pull request. Rientrano in gioco sub-agent (esplorare codebase) e skill (codificare convenzioni di progetto).

TDD come meccanismo di affidabilità

  • Si specifica il comportamento atteso come test prima di generare il codice.
  • Il test è contratto: l'agente sa quando ha finito.
  • Il test è feedback loop deterministico: l'agente sa se è sulla strada giusta.

Il ruolo umano si sposta su definizione del contratto e supervisione.
