Il loop agentico con tool

Terzo loop — l'agente decide cosa fare

Ad ogni turno il modello decide: rispondere all'utente, o emettere una tool call. Può emettere tool call multiple in sequenza prima di rispondere. Il numero di cicli non è predeterminato — lo decide il modello a runtime.

Limiti obbligatori in produzione: un tetto al numero di tool call per turno, un timeout per singola esecuzione, un budget di token per sessione. Senza, l'agente può entrare in cicli infiniti.
